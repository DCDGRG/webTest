## Project Overview

A React-based company website for Shanghai KuiXing Electronic Technology (上海奎星电子科技有限公司), a precision molding and special plastics manufacturing company. Built with React 19, TypeScript, Vite, and Bootstrap 5, featuring a multi-page SPA with industry news aggregation.

## Development Commands

### Essential Commands
- `npm run dev` - Start Vite development server (default port 5173)
- `npm run build` - TypeScript check (`tsc -b`) followed by production build
- `npm run lint` - Run ESLint with TypeScript and React configs
- `npm run preview` - Preview production build locally
- `npm run update-news` - Manually run news aggregator script (generates `public/data/news.json`)

### Testing
No test framework is currently configured in this project.

## Architecture

### Tech Stack
- **Framework**: React 19.1.1 with TypeScript 5.8
- **Build Tool**: Vite 7.1.2
- **Styling**: Bootstrap 5.3.7 + custom CSS (green theme)
- **Routing**: React Router DOM 7.8.0
- **Email**: EmailJS for contact form
- **News Aggregation**: RSS Parser with custom Node.js script

### Project Structure

```
src/
├── App.tsx              # Main app with BrowserRouter, Navbar, Footer, HomePage
├── main.tsx             # Entry point, imports CSS in specific order
├── pages/               # Route components (About, Contact, Pricing, BlogHome, etc.)
├── components/          # Reusable components (NewsCard, NewsSection)
├── types/               # TypeScript interfaces (News.ts)
└── *.css                # Multiple CSS files loaded in specific order

public/
├── data/
│   └── news.json       # Generated by aggregator script
└── logo.svg, favicon.jpg

scripts/
└── aggregator.js       # RSS feed aggregator with mock data
```

### Key Architectural Patterns

**CSS Loading Order** (critical for styling):
In `main.tsx`, CSS is imported in this exact order:
1. `styles.css` (Bootstrap/base)
2. `custom.css`
3. `green-theme.css`
4. `custom-green.css`
5. `bootstrap-icons/font/bootstrap-icons.css`

Do not reorder these imports.

**Routing Structure**:
- Single-page app with client-side routing via React Router
- Main navigation defined in `App.tsx` Navbar component
- Dropdown menus for "技术资讯" (Tech News) and "产品展示" (Portfolio)

**News System**:
- `scripts/aggregator.js` generates mock industry and technical news
- Outputs to `public/data/news.json`
- `NewsSection` component fetches and displays news
- GitHub Actions workflow runs aggregator daily at 00:00 UTC

## Development Guidelines

### Adding New Pages
1. Create component in `src/pages/`
2. Import in `App.tsx`
3. Add `<Route>` in `App.tsx` Routes
4. Add navigation link in Navbar or dropdown menus

### Modifying Styles
- Bootstrap utilities are available throughout
- Custom green theme colors defined in `green-theme.css` and `custom-green.css`
- Brand color: teal/green variants
- Common classes: `.light-teal-bg`, `.gradient-text`, `.elegant-shadow`, `.btn-cta`

### Working with News Data
- Mock data is hardcoded in `scripts/aggregator.js` (not real RSS feeds)
- Two categories: `industry` and `technical`
- When adding real RSS feeds, replace `MOCK_INDUSTRY_FEED` and `MOCK_TECHNICAL_FEED` in the SOURCES array
- News items follow the `NewsItem` interface in `src/types/News.ts`

### EmailJS Integration
- Contact form in `Contact.tsx` uses EmailJS
- Form fields: name (required), email (required), phone (optional), message (required)
- Service credentials are hardcoded - avoid committing changes to these values

## Important Notes

- **Chinese Content**: This is a Chinese-language website. Maintain Chinese text in UI components unless explicitly translating.
- **Bootstrap JavaScript**: Imported in `main.tsx` for navbar toggle functionality.
- **Vite Configuration**: Minimal configuration with only React plugin enabled.
- **No Test Suite**: Write tests manually if needed; no testing framework is configured.
- **GitHub Actions**: Automated news updates run daily via `.github/workflows/update_news.yml`.
